Differences between map,forEach,filter,find method of javascript

map

The map method creates a new array with the results of calling a callback function on every element in the original array1. It does not modify the original array. The callback function receives three arguments: the current element, the current index and the original array.

forEach

The forEach method executes a callback function for each element in the original array1. It does not return anything. It also does not modify the original array unless you explicitly do so inside the callback function. The callback function receives three arguments: the current element, the current index and the original array.

find

The find method returns the first element in the original array that satisfies a testing function2. It does not modify the original array. If no element passes the test, it returns undefined. The testing function receives three arguments: the current element,the current index and the original array.

filter

The filter method creates a new array with all the elements in the original array that pass a testing function. It does not modify the original array. The testing function receives three arguments: the current element, the current index and the original array.

What is Event Loop in javasript?

The event loop is a mechanism that allows JavaScript to execute code asynchronously and handle multiple events without blocking. It is responsible for executing the code, collecting and processing events, and executing queued sub-tasks.
The event loop consists of three main components:
A call stack that stores the functions that are currently being executed. The call stack operates on a last-in first-out (LIFO) principle: the last function pushed into the stack is the first one popped out when it returns.
A callback queue that stores the functions that are waiting to be executed after some asynchronous operation (such as a timer, a network request or a user interaction) has completed. The callback queue operates on a first-in first-out (FIFO) principle: the first function pushed into the queue is the first one popped out when it is ready to be executed. This means that the order of the callbacks in the queue reflects the order of their registration.
The browser API that provides various functions and objects that can perform asynchronous operations outside of the JavaScript engine. For example, the browser API includes the setTimeout function that can schedule a callback function to be executed after a certain delay. The browser API also includes the DOM manipulation methods, the XMLHttpRequest object and the fetch function that can make network requests.
